Đọc reporting views

📘 Khái niệm: VIEW trong PostgreSQL

View là bảng ảo (virtual table) được tạo từ câu lệnh SELECT phức tạp. Thay vì viết lại các phép JOIN nhiều bảng mỗi lần, ta đóng gói chúng thành 1 View và truy vấn như bảng thật. View không lưu dữ liệu — mỗi lần gọi sẽ chạy lại query gốc.

Hệ thống có 3 reporting views tổng hợp dữ liệu từ nhiều bảng: vw_student_progress_report (JOIN 5 bảng), vw_course_analytics (JOIN + GROUP BY + subquery), và vw_top_learners_leaderboard (JOIN + correlated subquery).

Báo cáo tiến độ học viên

vw_student_progress_report

Hiển thị khóa học, tiến độ và trạng thái học bằng cách đóng gói JOIN giữa user, profile, enrollment và course.

Xem CREATE VIEW: Tiến độ học viên 
CREATE OR REPLACE VIEW vw_student_progress_report AS
SELECT 
    up.full_name AS student_name,
    u.email,
    s.school_name,
    c.title AS course_title,
    ce.progress,
    ce.enrolled_at,
    CASE 
        WHEN ce.progress = 100 THEN 'Hoàn thành'
        WHEN ce.progress > 0 THEN 'Đang học'
        ELSE 'Mới đăng ký'
    END AS learning_status
FROM users u
JOIN user_profiles up ON u.user_id = up.user_id
JOIN students s ON u.user_id = s.user_id
JOIN course_enrollments ce ON s.user_id = ce.student_id
JOIN general_courses c ON ce.course_id = c.course_id
WHERE u.is_deleted = FALSE;

Phân tích khóa học

vw_course_analytics

Tổng hợp số học viên đăng ký, tiến độ trung bình và rating trung bình cho dashboard khóa học.

Xem CREATE VIEW: Phân tích khóa học 
CREATE OR REPLACE VIEW vw_course_analytics AS
SELECT 
    c.course_id,
    c.title AS course_title,
    up.full_name AS teacher_name,
    COUNT(ce.enrollment_id) AS total_students,
    ROUND(AVG(ce.progress), 2) AS avg_progress,
    (
        SELECT ROUND(AVG(rating), 1)
        FROM user_feedbacks
        WHERE context LIKE '%' || c.title || '%'
    ) AS avg_rating
FROM general_courses c
JOIN teachers t ON c.teacher_id = t.user_id
JOIN user_profiles up ON t.user_id = up.user_id
LEFT JOIN course_enrollments ce ON c.course_id = ce.course_id
WHERE c.is_deleted = FALSE
GROUP BY c.course_id, c.title, up.full_name;

Bảng xếp hạng học viên

vw_top_learners_leaderboard

Xếp hạng học viên theo streak và số achievement, người demo không phải viết lại subquery.

Xem CREATE VIEW: Bảng xếp hạng 
CREATE OR REPLACE VIEW vw_top_learners_leaderboard AS
SELECT 
    up.full_name,
    up.avatar_url,
    ss.current_streak,
    ss.highest_streak,
    (
        SELECT COUNT(*)
        FROM user_achievements ua
        WHERE ua.user_id = s.user_id
    ) AS total_achievements
FROM students s
JOIN user_profiles up ON s.user_id = up.user_id
JOIN student_streaks ss ON s.user_id = ss.student_id
ORDER BY ss.current_streak DESC, total_achievements DESC;
Điều gì xảy ra trong database? Mỗi nút chỉ chạy một view, giúp trình bày từng báo cáo riêng thay vì gộp cả 3 vào một hành động.
TRIGGER SESSION 1

Trigger tự động cập nhật UpdatedAt

⚡ Khái niệm: TRIGGER trong PostgreSQL

Trigger là hàm tự động thực thi khi có sự kiện INSERT/UPDATE/DELETE xảy ra trên bảng. BEFORE trigger chạy trước khi dữ liệu thay đổi (có thể sửa NEW), AFTER trigger chạy sau khi thay đổi đã commit. Trigger đảm bảo business rules tự động mà không cần code ở tầng application.

Khi UPDATE bất kỳ cột nào trên bảng users, trigger trg_users_update_timestamp (BEFORE UPDATE) sẽ tự động cập nhật cột updated_at = CURRENT_TIMESTAMP. Quan sát giá trị updated_at TRƯỚC và SAU.

📄 Xem mã SQL (CREATE TRIGGER) 
-- Trigger Function: tự động cập nhật updated_at
CREATE OR REPLACE FUNCTION fn_update_timestamp()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
BEGIN
    NEW.updated_at := CURRENT_TIMESTAMP;
    RETURN NEW;  -- BEFORE trigger phải RETURN NEW
END;
$$;

-- Gắn trigger vào bảng users (BEFORE UPDATE)
CREATE TRIGGER trg_users_update_timestamp
    BEFORE UPDATE ON users
    FOR EACH ROW
    EXECUTE FUNCTION fn_update_timestamp();

-- Giảng viên chú ý: BEFORE UPDATE → sửa NEW trước khi ghi
-- FOR EACH ROW → chạy cho mỗi dòng bị UPDATE
🔄 BEFORE UPDATE → tự gán NEW.updated_at
Điều gì xảy ra trong database? Câu UPDATE chỉ đổi username. BEFORE UPDATE trigger tự gán NEW.updated_at trước khi PostgreSQL ghi dòng mới.
TRIGGER SESSION 2

Trigger tạo streak học viên

Khi INSERT một student mới vào bảng students, trigger trg_after_insert_student (AFTER INSERT) tự động tạo 1 dòng tương ứng trong bảng student_streaks với streak = 0. Đảm bảo tính toàn vẹn dữ liệu — mọi student luôn có streak record.

📄 Xem mã SQL (CREATE TRIGGER) 
-- Trigger Function: tự động tạo streak record
CREATE OR REPLACE FUNCTION fn_init_student_streak()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
BEGIN
    INSERT INTO student_streaks (student_id, current_streak, highest_streak)
    VALUES (NEW.user_id, 0, 0);
    RETURN NEW;  -- AFTER trigger: RETURN không ảnh hưởng
END;
$$;

-- Gắn trigger vào bảng students (AFTER INSERT)
CREATE TRIGGER trg_after_insert_student
    AFTER INSERT ON students
    FOR EACH ROW
    EXECUTE FUNCTION fn_init_student_streak();

-- Giảng viên chú ý: AFTER INSERT → chạy SAU khi INSERT thành công
-- NEW.user_id → lấy ID của student vừa được insert
➕ AFTER INSERT → tự tạo student_streaks row
TRIGGER SESSION 3

Trigger chặn publish khóa học rỗng

Trigger trg_before_publish_course (BEFORE UPDATE) kiểm tra: nếu khóa học chuyển từ DRAFT → PUBLISHED mà chưa có module nào, trigger sẽ RAISE EXCEPTION → PostgreSQL tự động ROLLBACK thao tác. Demo chạy 2 bước: thử publish khóa rỗng bị chặn, thêm module rồi publish lại thành công.

📄 Xem mã SQL (CREATE TRIGGER) 
-- Trigger Function: chặn publish khóa học rỗng
CREATE OR REPLACE FUNCTION fn_validate_course_publish()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
DECLARE
    v_module_count INTEGER;
BEGIN
    -- Chỉ kiểm tra khi chuyển sang PUBLISHED
    IF NEW.visibility_status = 'PUBLISHED'
       AND (OLD.visibility_status IS DISTINCT FROM 'PUBLISHED') THEN
        SELECT COUNT(*) INTO v_module_count
        FROM general_course_modules
        WHERE course_id = NEW.course_id;

        IF v_module_count = 0 THEN
            RAISE EXCEPTION 'Cannot publish: course has no modules';
        END IF;
    END IF;
    RETURN NEW;
END;
$$;

-- Gắn trigger (BEFORE UPDATE)
CREATE TRIGGER trg_before_publish_course
    BEFORE UPDATE ON general_courses
    FOR EACH ROW
    EXECUTE FUNCTION fn_validate_course_publish();

-- Giảng viên chú ý: RAISE EXCEPTION → abort transaction
-- BEFORE UPDATE + RETURN NEW → cho phép hoặc chặn UPDATE
🛡️ BEFORE UPDATE → RAISE EXCEPTION nếu vi phạm
Điều gì xảy ra trong database? Khi visibility_status chuyển sang PUBLISHED, trigger đếm module của khóa học. Nếu module_count = 0, PostgreSQL raise exception và rollback UPDATE.
